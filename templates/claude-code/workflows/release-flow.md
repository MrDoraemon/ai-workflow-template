# Release Workflow / 发布流水线

## 触发条件
用户要求发布或部署。

## 编排流程

### Phase 1: 预检
1. 调用 qa 运行全量测试
2. 调用 security 做只读安全审计
3. 如需运行依赖审计或安全扫描命令，由主会话或具备执行权限的 Agent 在用户确认后执行
4. 确认无致命/严重问题（如有则中止，回到修复流程）

### Phase 2: 构建
1. 调用 devops
   - 项目生产构建或打包
   - 容器镜像构建（如有 Dockerfile）
   - 配置验证
2. 输出构建产物和构建状态

### Phase 3: 部署（需用户确认）
1. **人工门控点**：展示构建结果和部署计划，等待用户确认
2. devops 执行部署流程
3. 输出部署状态和验证结果

## 关键约束
- Phase 1 的致命/严重问题必须修复后才能继续
- Phase 3 部署前必须获得用户确认
- 生产环境 Dockerfile 必须使用非 root 用户
- 禁止在镜像中包含 `.env` 文件
